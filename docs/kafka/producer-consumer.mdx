## 1. Zookeeper & Kafka Broker setup

Zookeeper 와 Kafka Broker 를 띄우는 `docker-compose.yaml` 파일을 작성해보도록 하겠습니다.

### 1.1 Zookeeper service

먼저, Zookeeper service 를 띄울 때 쓰이는 요소들에 대해 알아보도록 하겠습니다.

<CodeDescription>

```yaml  title="docker-compose.yaml"
version: "3"

services:
	zookeeper:
    image: confluentinc/cp-zookeeper:7.3.0
    container_name: zookeeper
    ports:
      - 2181:2181
    environment:
      ZOOKEEPER_SERVER_ID: 1
      ZOOKEEPER_CLIENT_PORT: 2181
```
- `image` : zookeeper 를 띄울 때 쓰이는 image 를 명시합니다. 여기서는 `confluentinc/cp-zookeeper:7.3.0` 를 이용하도록 하겠습니다.
- `container_name` : container 의 이름을 지정합니다. 여기서는 `zookeeper` 로 사용하겠습니다.
- `ports` : zookeeper 의 port 를 지정합니다. 여기서는 `2181:2181` 을 사용하겠습니다.
- `environment` : zookeeper 의 환경 변수를 지정합니다.
    - `ZOOKEEPER_SERVER_ID` : zookeeper cluster 에서 해당 zookeeper 를 식별할 ID 를 지정합니다. 여기서는 ID 를 1로 지정하겠습니다.
    - `ZOOKEEPER_CLIENT_PORT` : zookeeper client 의 port 를 지정합니다. container 내부에서 zookeeper 는 지정된 port 로 실행됩니다. 여기서는 기본 zookeeper 의 port 인 2181 로 지정하겠습니다.

</CodeDescription>

### 1.2 Kafka Broker service

다음으로 Kafka Broker service 를 띄울 때 쓰이는 요소들에 대해 알아보도록 하겠습니다. 여기서는 single broker 를 가정하고 작성해보겠습니다.

<CodeDescription>

```yaml  title="docker-compose.yaml"
broker:
    image: confluentinc/cp-kafka:7.3.0
    container_name: broker
    depends_on:
      - zookeeper
    ports:
      - 9092:9092
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://broker:29092,PLAINTEXT_HOST://localhost:9092
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
      KAFKA_INTER_BROKER_LISTENER_NAME: PLAINTEXT
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
      KAFKA_GROUP_INITIAL_REBALANCE_DELAY_MS: 0
```
- `image` : broker 를 띄울 때 쓰이는 image 를 명시합니다. 여기서는 `confluentinc/cp-kafka:7.3.0` 를 이용하도록 하겠습니다.
- `container_name` : container 의 이름을 지정합니다. 여기서는 `broker` 로 사용하겠습니다.
- `depends_on` : service 간의 종속성을 설정합니다. 여기서는 `zookeeper` 가 먼저 띄워진 다음에 broker 띄워져야함으로 `zookeeper` 를 적어줍니다.
- `ports` : broker 의 port 를 지정합니다. 여기서는 `9092:9092` 을 사용하겠습니다.
- `environment` : broker 의 환경 변수를 지정합니다.
    - `KAFKA_SERVER_ID` : broker 의 ID 를 지정합니다. Single broker 에서는 없어도 무방하나 여기서는 1로 지정하겠습니다.
    - `KAFKA_ZOOKEEPER_CONNECT` : broker 가 zookeeper 에 connection 하기 위한 대상을 지정합니다. 여기서는 `<zookeeper service name>:<zookeeper service port>` 인 `zookeeper:2181` 을 사용하겠습니다.
    - `KAFKA_ADVERTISED_LISTENERS` : 내부와 외부에서 접속하기 위한 리스너를 설정합니다. 일반적으로 internal 과 external 두 가지를 설정합니다. 여기서는 internal 로 `PLAINTEXT://broker:29092` 로 설정하고, external 로 `PLAINTEXT_HOST://localhost:9092` 으로 설정하겠습니다. 최종적으로 `PLAINTEXT://broker:29092,PLAINTEXT_HOST://localhost:9092` 을 사용하겠습니다.
    - `KAFKA_LISTENER_SECURITY_PROTOCOL_MAP` : 보안을 위한 protocol mapping을 설정합니다. 이 설정값은 `KAFKA_ADVERTISED_LISTENERS` 과 함께 key/value 로 매핑됩니다. 여기서는 `PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT` 로 설정하겠습니다.
    - `KAFKA_INTER_BROKER_LISTENER_NAME` : container 내부에서 사용할 리스너 이름을 지정합니다. 여기서는 앞서 internal 로 설정했던 `PLAINTEXT` 을 사용하겠습니다.
    - `KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR` : topic 을 분산하여 저장할 replication factor 를 설정합니다. 여기서는 single broker 를 사용하기 때문에 1로 지정하겠습니다. multi broker 의 경우, 기본값인 3을 사용하므로 이 설정을 넣을 필요는 없습니다.
    - `KAFKA_GROUP_INITIAL_REBALANCE_DELAY_MS` : kafka cluster 에서 초기에 rebalancing 할 때  consumer 들이 consumer group 에 조인할 때 대기하는 시간입니다. 여기서는 0으로 설정해줍니다.

</CodeDescription>


### 1.3 전체 코드

최종적인 `docker-compose.yaml` 구성은 아래와 같습니다.

```yaml  title="docker-compose.yaml"

services:
  zookeeper:
    image: confluentinc/cp-zookeeper:7.3.0
    container_name: zookeeper
    ports:
      - 2181:2181
    environment:
      ZOOKEEPER_SERVER_ID: 1
      ZOOKEEPER_CLIENT_PORT: 2181
  broker:
    image: confluentinc/cp-kafka:7.3.0
    container_name: broker
    depends_on:
      - zookeeper
    ports:
      - 9092:9092
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://broker:29092,PLAINTEXT_HOST://localhost:9092
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
      KAFKA_INTER_BROKER_LISTENER_NAME: PLAINTEXT
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
      KAFKA_GROUP_INITIAL_REBALANCE_DELAY_MS: 0
```

### 1.4 실행

아래 명령어를 활용해 정의된 서비스를 백그라운드 프로세스로 띄웁니다.

## 2. Producer & Consumer setup

이번에는 topic 을 생성해보고, 이어서 producer 와 consumer 를 생성해보도록 하겠습니다.

### 2.1 Topic 생성

먼저 topic 을 생성해보겠습니다.

<CodeDescription>

```bash
# terminal-command
docker compose exec broker kafka-topics --create --topic topic-test --bootstrap-server broker:29092 --partitions 1 --replication-factor 1
```

- `docker compose exec` : container 내에 명령어를 수행합니다.
- `broker` : 생성된 broker service 의 이름을 적습니다.
- `kafka-topics` : kafka topic 에 대한 명령을 실행합니다.
- `--create` : topic 을 생성합니다.
- `--topic` : 생성할 topic 의 이름을 지정합니다. 여기서는 `topic-test` 라는 이름으로 생성해보겠습니다.
- `--bootstrap-server` : broker service 에 대한 hostname 와 port 를  지정합니다. 여기서는 앞서 docker-compose 로 띄웠던 broker 의 환경 변수를 참고하여 `broker:29092` 로 설정합니다.
- `--partition` : topic 내에 partition 개수를 설정합니다. 여기서는 1로 설정하겠습니다.
- `--replication-factor` : replication factor 를 지정합니다. 여기서는 1로 설정하겠습니다.

</CodeDescription>

이어서 topic 이 잘 생성되었는지 확인해보도록 하겠습니다.

<CodeDescription>

```bash
# terminal-command
docker compose exec broker kafka-topics --describe --topic topic-test --bootstrap-server broker:29092
```

- `--describe` :  생성된 topic 에 대한 상세 설명을 보여줍니다.

</CodeDescription>

확인하면 다음과 같이 출력됩니다.

```bash
Topic: topic-test       TopicId: pcR8cByuSVG1p1guQ3MFRg PartitionCount: 1       ReplicationFactor: 1   Configs: 
        Topic: topic-test       Partition: 0    Leader: 1       Replicas: 1     Isr: 1
```

### 2.1 Consumer 실행

이제 topic 을 다 만들었으니 consumer 를 실행하도록 하겠습니다. Consumer 를 먼저 실행하는 이유는 일반적으로 consumer 가 메시지를 subscribe 하려고 대기하는 상태에서 producer 가 메시지를 생성해서 보내기 때문입니다.

먼저 `docker compose exec` 명령어를 통해 container 내부로 접속합니다.

```bash
# terminal 1
# terminal-command
docker compose exec broker bash
```

접속되면 다음과 같이 터미널이 열립니다.

```bash
# terminal-command
docker compose exec broker bash
[appuser@0a0599db4d96 ~]$
```

이후에 `kafka-console-consumer` 를 이용하여 `topic-test` topic 을 subscribe 합니다.

<CodeDescription>

```bash  title="[appuser@0a0599db4d96 ~]$"
# terminal-command
kafka-console-consumer --topic topic-test --bootstrap-server broker:29092
```
- Topic 을 생성했던 것처럼 bootstrap-server 를 `broker:29092` 로 설정합니다.

</CodeDescription>

실행하면 다음과 같이 수신을 대기하고 있는 상태가 됩니다.

```bash
[appuser@0a0599db4d96 ~]$ kafka-console-consumer --topic topic-test --bootstrap-server broker:29092

```

### 2.2 Producer 실행

마지막으로 producer 를 만들어서 메시지를 보낼 준비를 하겠습니다.

Consumer 와 똑같이, `docker compose exec` 명령어를 통해 container 내부로 접속합니다. 이 때 consumer 를 실행했던 terminal 이 아닌 새로운 terminal 로 띄웁니다.

```bash
# terminal 2
# terminal-command
docker compose exec broker bash
```

접속되면 다음과 같이 터미널이 열립니다.

```bash
# terminal-command
docker compose exec broker bash
[appuser@0a0599db4d96 ~]$ 
```

이후에 `kafka-console-producer` 를 이용하여 `topic-test` topic 에 접근하여 publish 할 준비를 합니다.

```bash  title="[appuser@0a0599db4d96 ~]$ "
# terminal-command
kafka-console-producer --topic topic-test --broker-list broker:29092
```

명령어를 실행하면 다음과 같이 publish 할 수 있는 상태가 됩니다.

```bash
[appuser@0a0599db4d96 ~]$ kafka-console-producer --topic topic-test --broker-list broker:29092
> 
```

### 2.3 Producer 에 메시지 입력하기

아래와 같이 `hello`, `this is mlops for mle` 를 입력해봅니다.

```bash
[appuser@0a0599db4d96 ~]$ kafka-console-producer --topic topic-test --broker-list broker:29092
> hello
> this is mlops for mle
```
